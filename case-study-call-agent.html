FAQ

Common Questions

Everything you need to know before deploying the agent for your clinic.

Still have questions? Book a call →
What if a patient's preferred doctor is fully booked?
▼
The agent scans forward in 30-minute increments for the same doctor on the same day. If nothing's available before closing, it informs the patient - it does not silently reassign to another doctor.
Can two patients accidentally book the same slot?
▼
Both requests check the live database before inserting. Simultaneous requests on the exact same slot are a known edge case worth discussing if your clinic is very high-volume.
What gets stored and where does the code come from?
▼
A random 6-character alphanumeric code is generated per booking. The record stores patient name, phone, doctor, date, time slot, code, and timestamp. The code is sent via SMS immediately after confirmation.
How does cancellation work?
▼
The patient provides their appointment code. The agent matches it against the database - if found, the record is deleted and an SMS confirms the cancellation. Only the code is validated, not the name.
